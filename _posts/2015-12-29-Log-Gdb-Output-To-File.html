<p>Sometimes, especially when you&#8217;re dealing with a long stack trace, or a multi-threaded stack trace, trying to view the debug output from GDB in a terminal window can be a little awkward.</p> <p>Did you know that you can log specific output (or even the entire session) from within GDB to a text file?</p> <p>All you have to do is type:</p> <pre>set logging on</pre> <p>at the GDB prompt.</p> <p>GDB saves all output from this point in a text file called gdb.txt that resides in the directory in which you are running GDB (usually the same directory as your executable).</p> <p>When you&#8217;re done you can type:</p> <pre>set logging off</pre> <p>And GDB will stop recording output.</p> <p>Note that you can turn logging on and off several times and GDB will concatenate output to the gdb.txt file, so you don&#8217;t have to worry about overwriting what you&#8217;ve already got there.</p> <p>If you want to record multiple files in a session you can change the filename with:</p> <pre>set logging file myfile.txt</pre> <p>Here&#8217;s a little screenshot:</p> <p><img class="alignnone wp-image-2816 size-full" src="" alt="gdb-file-logging" width="575" height="251" /></p> </div> #+END_HTML
